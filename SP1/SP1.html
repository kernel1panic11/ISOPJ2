Sprint 2. Administración de almacenamiento, cuotas y gestión básica de usuarios en Windows 11

En este segundo sprint se ha comenzado a trabajar sobre un sistema Windows 11 ya instalado y operativo, orientando la práctica a la administración del almacenamiento, la gestión de usuarios y grupos y el control del uso de espacio mediante cuotas de disco. Hasta este punto se ha completado la preparación del disco adicional, la creación de particiones con distintos sistemas de archivos, la verificación de la estructura de volúmenes, la activación de cuotas sobre NTFS y la comprobación práctica de su funcionamiento con un usuario real. Esta secuencia encaja con la propuesta del sprint, que plantea primero la preparación del sistema, después la creación de cuotas y usuarios, y más adelante la automatización, verificación y permisos avanzados.

Fase 1. Preparación del almacenamiento

El primer paso consistió en añadir a la máquina virtual un nuevo disco de 20 GB en formato NVMe. Este disco se incorporó como almacenamiento independiente del disco principal del sistema, de modo que todas las pruebas de particionado y gestión de volúmenes se realizaran sobre un entorno separado del disco del sistema operativo. Una vez detectado por Windows, se abrió el administrador de discos y se inicializó el nuevo dispositivo utilizando el esquema de partición GPT. Esta elección es coherente con un sistema actual como Windows 11, ya que GPT es el esquema moderno recomendado para entornos UEFI y ofrece una estructura de particionado más robusta que MBR.

Sobre ese nuevo disco se creó una primera partición principal de aproximadamente 15 GB en NTFS, denominada Dades, y una segunda partición de aproximadamente 5 GB en FAT32, denominada Portable. Esta distribución responde al uso previsto de cada volumen. La partición Dades, al estar en NTFS, se reserva para las tareas que requieren soporte de cuotas y control avanzado del sistema de archivos. La partición Portable, en cambio, se mantiene en FAT32 para disponer de un volumen alternativo con un sistema de archivos distinto, lo que permite comparar características y limitaciones entre ambos formatos, tal como también se indica en el enunciado general del sprint.

Una vez creados los volúmenes, se verificó la configuración desde línea de comandos utilizando diskpart. Con list disk se confirmó la presencia de dos discos, uno principal de 64 GB y el nuevo disco secundario de 20 GB. Después, con list volume, se comprobaron las particiones visibles en el sistema, incluyendo los nuevos volúmenes Dades y Portable con sus respectivas letras y tamaños. Esta validación permitió confirmar que la estructura de almacenamiento había quedado correctamente reconocida por Windows antes de avanzar hacia la configuración de cuotas. La comprobación mediante diskpart aparece expresamente en la propuesta del sprint como parte del cierre de esta primera fase.

Fase 2. Cuotas de disco, usuarios y grupos

Una vez preparada la partición Dades, se accedió a sus propiedades para revisar la pestaña de cuotas. En un primer momento, la gestión de cuotas se encontraba sin configurar. Posteriormente se habilitó la administración de cuotas y se definió un límite de 300 MB por usuario, junto con un nivel de advertencia previo. Esta configuración convierte a la partición NTFS en un volumen controlado, capaz de restringir el espacio disponible por identidad. Desde el punto de vista técnico, este comportamiento diferencia claramente a NTFS de FAT32, ya que solo el primero permite aplicar este tipo de control de almacenamiento por usuario. La propuesta del sprint establece precisamente la activación de cuotas sobre la partición de datos y la fijación de un límite con advertencia como parte central de esta fase.

A continuación se crearon dos usuarios locales, alumne1 y alumne2, que serán utilizados en el resto del sprint para probar comportamientos diferenciados sobre almacenamiento y permisos. Después se creó el grupo Limitats y se añadieron ambos usuarios a dicho grupo. Esta organización permite administrar el acceso de forma más ordenada, dejando preparada la base para futuras configuraciones de permisos sobre carpetas y recursos compartidos. El enunciado general insiste precisamente en la creación de cuentas locales y grupos como parte de la administración del sistema, y la propuesta del sprint concreta estos nombres y esta agrupación.

Para comprobar el funcionamiento real de las cuotas, se inició sesión con el usuario alumne1 y se realizaron pruebas directas sobre la unidad Dades. Tras el primer inicio de sesión, se generaron archivos mediante línea de comandos con el objetivo de consumir espacio en la partición limitada. El resultado observado fue que la primera creación se completaba correctamente, mientras que la segunda ya fallaba al superarse el umbral configurado. Esto confirma que la cuota aplicada estaba actuando de forma efectiva y que el sistema restringía la escritura una vez alcanzado el límite definido para ese usuario. La propuesta del sprint señala precisamente esta prueba de copia de ficheros como la forma de verificar el comportamiento de las cuotas al superar el límite establecido.

Con estas acciones queda documentada la parte realizada hasta ahora del Sprint 2. El entorno dispone ya de almacenamiento adicional correctamente estructurado, con dos sistemas de archivos diferenciados, una partición NTFS preparada para administración avanzada, cuotas activas y usuarios reales sobre los que validar su funcionamiento. A partir de este punto, el sistema queda listo para continuar con las siguientes fases previstas, como la automatización de copias, la gestión de procesos y la aplicación de permisos avanzados mediante ACL.
