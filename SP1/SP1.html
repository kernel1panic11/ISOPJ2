Sprint 1. Instalación y configuración inicial de Windows

Descripción general

Esta práctica documenta la puesta en marcha inicial de un entorno Windows en máquina virtual, desde la creación de la VM hasta la comprobación de herramientas básicas del sistema. El contenido se ha reorganizado a partir de la guía paso a paso del Sprint 1 para que el flujo de trabajo quede claro y siga el orden real de la práctica.

El recorrido se divide en siete bloques: instalación del sistema, puntos de restauración, licenciamiento, gestor de arranque, red básica, comandos generales e instalación de aplicaciones. Cada fase deja preparado el sistema para la siguiente y permite validar que Windows queda operativo y correctamente configurado.

Fase 1. Creación de la máquina virtual e instalación de Windows

El primer bloque consiste en crear la máquina virtual en VirtualBox y asignarle una base de recursos suficiente para trabajar con Windows sin cuellos de botella. La guía marca como referencia un mínimo de 4 GB de RAM y un disco virtual de al menos 40 GB, lo que permite instalar el sistema y seguir practicando con margen para nuevas configuraciones.

  1. Crear la máquina virtual y asignar memoria RAM, CPU y almacenamiento inicial.
  2. Cargar la ISO de Windows 10 o Windows 11 como medio de instalación.
  3. Completar el asistente de instalación indicando idioma, usuario y contraseña.
  4. Finalizar el despliegue y comprobar que el sistema arranca correctamente después del primer reinicio.

Esta fase no se limita a instalar Windows: también deja documentado que la máquina virtual queda funcional, accesible y lista para recibir ajustes posteriores de seguridad, mantenimiento y administración.

Fase 2. Puntos de restauración

Una vez instalado el sistema, la siguiente tarea es activar la protección del sistema y trabajar con puntos de restauración. El objetivo es disponer de un mecanismo de recuperación rápida antes de hacer cambios de configuración o instalar software adicional.

  1. Buscar la opción Crear un punto de restauración en Windows.
  2. Activar la protección del sistema sobre la unidad C:.
  3. Crear un punto de restauración manual.
  4. Realizar un cambio en el sistema, como instalar una aplicación o modificar una configuración.
  5. Restaurar el sistema al punto anterior y comprobar que el cambio queda revertido.

Este bloque deja justificado el uso del servicio de restauración como medida básica de seguridad operativa: se prueba tanto la creación como la recuperación real del estado del equipo.

Fase 3. Licenciamiento y activación de Windows

La práctica continúa revisando el estado de activación del sistema. No basta con instalar Windows; también hay que identificar si el sistema está activado, qué tipo de licencia utiliza y cómo se puede verificar desde la propia interfaz y desde línea de comandos.

  1. Abrir Configuración > Sistema > Activación y revisar el estado de Windows.
  2. Ejecutar en consola el comando slmgr /xpr para comprobar la vigencia de la licencia.
  3. Identificar el tipo de licenciamiento del sistema y explicarlo brevemente.
  4. Consultar el precio aproximado de una licencia de Windows en una fuente oficial o comercial reconocida.

Esta fase combina observación técnica y explicación conceptual, ya que obliga a interpretar qué significa que un sistema esté activado y cómo se valida ese estado en un entorno real.

Fase 4. Gestor de arranque

En el cuarto bloque se analiza el arranque del sistema mediante bcdedit. La finalidad es entender cómo Windows organiza la información del Boot Manager y del Boot Loader, y qué datos son relevantes para interpretar el proceso de inicio.

  1. Abrir Command Prompt como administrador.
  2. Ejecutar bcdedit.
  3. Identificar el bloque Windows Boot Manager.
  4. Identificar el bloque Windows Boot Loader.
  5. Interpretar valores como default, timeout, device, path y description.

Las preguntas clave de esta fase son: qué sistema arranca por defecto, en qué partición está instalado, cuánto espera el gestor antes de iniciar y qué fichero participa en la carga del sistema. La conclusión esperada es diferenciar claramente quién decide el arranque y quién carga Windows. 

bcdedit
slmgr /xpr

Fase 5. Red básica

Después del arranque se revisa la conectividad de red. La práctica exige comprobar tanto la configuración automática por DHCP como una configuración manual con IP fija, máscara, puerta de enlace y DNS.

  1. Abrir la configuración de red del sistema.
  2. Consultar la IP actual con ipconfig.
  3. Configurar la interfaz en modo dinámico mediante DHCP.
  4. Configurar la interfaz en modo estático con IP, máscara, gateway y DNS.
  5. Verificar conectividad con ping google.com.

El valor de esta fase está en comparar ambos escenarios de configuración y comprobar que el sistema mantiene conectividad después de cada cambio.

Fase 6. Comandos generales del sistema

El sexto bloque introduce el uso básico de cmd y PowerShell. La guía separa ambos entornos para remarcar que el símbolo del sistema está orientado a comandos clásicos, mientras que PowerShell permite automatización más potente y trabajo con objetos.

Comandos básicos que se deben probar en la práctica:

dir
cd
mkdir prueba
echo hola > fitxer.txt
del fitxer.txt

Comandos útiles de sistema y red revisados en esta fase:

tasklist
taskkill /IM notepad.exe /F
systeminfo
hostname
whoami
ipconfig
ping google.com
netstat -an
tree
cls
help
shutdown /s /t 0

Además de ejecutarlos, la guía pide interpretar su salida: qué muestra tasklist, qué información aporta ipconfig y qué datos generales del equipo ofrece systeminfo. Ese análisis es parte de la documentación, no un añadido opcional.

Fase 7. Instalación y desinstalación de aplicaciones

La fase final se centra en la instalación de software desde distintas vías y en la verificación posterior de su funcionamiento. La práctica no termina al abrir el asistente: también exige confirmar que la aplicación queda disponible en el sistema y que su desinstalación elimina su presencia funcional.

  1. Descargar una aplicación desde el navegador, por ejemplo Chrome o VS Code.
  2. Instalarla con el asistente y comprobar que abre correctamente.
  3. Instalar otra aplicación desde Microsoft Store y verificar su funcionamiento.
  4. Desinstalar una aplicación desde Configuración > Aplicaciones.
  5. Comprobar que el programa ya no aparece ni queda operativo en el sistema.

Conclusión

El Sprint 1 deja un entorno Windows completamente preparado para prácticas posteriores. El sistema queda instalado, verificado, con mecanismos básicos de recuperación, revisión de licencia, interpretación del arranque, conectividad funcional, manejo de consola y control sobre la instalación de aplicaciones. Ese punto de partida encaja con lo que necesita el resto del proyecto: una base estable sobre la que construir prácticas de administración más avanzadas.
