Unidad 1. Introducción a los Sistemas Operativos y Planificación

Volver al inicio Siguiente unidad

Guía de documentación

Esta unidad está preparada para documentarse con capturas de pantalla del proceso completo en VMware. La idea es añadir evidencias visuales de cada fase y, siempre que sea posible, una captura por paso importante para mostrar el procedimiento de forma ordenada.

Fase 1. Instalación del sistema operativo

  1. Crear una máquina virtual con VMware.
  2. Asignar recursos: RAM mínima de 4 GB y disco mínimo de 40 GB.
  3. Cargar la ISO de Windows 10 o Windows 11.
  4. Instalar el sistema configurando idioma, usuario y contraseña.
  5. Comprobar que el sistema arranca correctamente.

Capturas recomendadas: creación de la máquina virtual, asignación de recursos, carga de la ISO, asistente de instalación y primer arranque correcto.

Fase 2. Puntos de restauración

  1. Buscar la opción Crear un punto de restauración.
  2. Activar la protección del sistema en el disco C:.
  3. Crear un punto de restauración manual.
  4. Realizar un cambio en el sistema, como instalar una aplicación o modificar una configuración.
  5. Restaurar el sistema y comprobar el resultado.

Capturas recomendadas: acceso a la herramienta, activación en C:, creación del punto, cambio realizado y comprobación final tras la restauración.

Fase 3. Licencias de Windows

  1. Abrir Configuración > Sistema > Activación.
  2. Comprobar si Windows está activado.
  3. Ejecutar en cmd el comando slmgr /xpr.
  4. Averiguar el tipo de licenciamiento de Windows y explicarlo brevemente.
  5. Consultar el precio aproximado de una licencia de Windows.

Capturas recomendadas: pantalla de activación, resultado de slmgr /xpr y referencia del precio consultado.

Fase 4. Gestor de arranque

  1. Abrir el Símbolo del sistema como administrador.
  2. Ejecutar el comando bcdedit.
  3. Identificar los bloques:
    • Administrador de arranque de Windows (Boot Manager).
    • Cargador de arranque de Windows (Boot Loader).
  4. Interpretar los datos concretos:
    • En el bloque Boot Manager, identificar default {current} como el sistema que arranca por defecto.
    • En el bloque Boot Manager, identificar timeout 30 como el tiempo de espera antes del arranque.
    • En el bloque Boot Loader, identificar device partition=C: como la partición donde está instalado Windows.
    • En el bloque Boot Loader, identificar \Windows\system32\winload.efi como el archivo que carga el sistema.
    • En el bloque Boot Loader, identificar description Windows 11 como el sistema operativo detectado.
  5. Responder a las siguientes preguntas:
    • Qué sistema se está arrancando.
    • En qué disco o partición está instalado.
    • Cuánto tiempo espera antes de arrancar.
    • Qué archivo inicia Windows.
  6. Realizar una interpretación final en una frase:
    • Quién decide el arranque: el Boot Manager.
    • Quién carga el sistema: el Boot Loader.

Capturas recomendadas: consola abierta como administrador, salida de bcdedit y detalle de los bloques Boot Manager y Boot Loader.

Fase 5. Red básica

  1. Abrir la configuración de red.
  2. Consultar la IP con ipconfig.
  3. Configurar una IP dinámica usando DHCP automático.
  4. Configurar una IP fija de forma manual indicando IP, máscara, gateway y DNS.
  5. Comprobar la conexión con ping google.com.

Capturas recomendadas: configuración de red, salida de ipconfig, configuración DHCP, configuración manual y prueba de conectividad.

Fase 6. Comandos generales

  1. Abrir PowerShell.
  2. Diferenciar entre cmd y PowerShell:
    • cmd se usa para comandos básicos.
    • PowerShell es más potente y está orientado a automatización.
  3. Usar comandos básicos:
    • dir para ver archivos.
    • cd para moverse entre carpetas.
    • mkdir prueba para crear una carpeta.
    • echo hola > fitxer.txt para crear un archivo.
    • del fitxer.txt para eliminar el archivo.
  4. Usar comandos útiles:
    • tasklist para ver procesos.
    • taskkill /IM notepad.exe /F para cerrar un proceso.
    • systeminfo para consultar información del sistema.
    • hostname para ver el nombre del equipo.
    • whoami para saber el usuario actual.
  5. Usar comandos de red:
    • ipconfig para ver la configuración IP.
    • ping google.com para comprobar conectividad.
    • netstat -an para ver conexiones abiertas.
  6. Usar comandos avanzados:
    • tree para ver la estructura de carpetas.
    • cls para limpiar la pantalla.
    • help para obtener ayuda.
    • shutdown /s /t 0 para apagar el equipo.
  7. Hacer una mini interpretación indicando qué muestra:
    • tasklist.
    • ipconfig.
    • systeminfo.

Capturas recomendadas: ejecución en PowerShell, comandos básicos, comandos de red, comandos avanzados y resultados de tasklist, ipconfig y systeminfo.

Fase 7. Instalación de aplicaciones

  1. Descargar un programa, por ejemplo Chrome o Visual Studio Code.
  2. Instalarlo.
  3. Abrirlo y comprobar que funciona correctamente.
  4. Instalar una aplicación desde Microsoft Store.
  5. Abrirla y comprobar su funcionamiento.
  6. Desinstalar una aplicación.
  7. Verificar que ya no aparece instalada.

Capturas recomendadas: descarga del instalador, instalación, ejecución, instalación desde Microsoft Store, comprobación y desinstalación final.
