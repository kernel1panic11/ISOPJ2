Sprint 1. Instalación y configuración inicial de Windows 11 en máquina virtual

En este sprint se ha realizado la implantación inicial de Windows 11 sobre una máquina virtual en VMware, partiendo desde la creación del entorno virtual hasta la validación operativa del sistema ya instalado. El trabajo llevado a cabo incluye la preparación de la máquina virtual, la instalación del sistema operativo, la configuración inicial del entorno, la instalación de VMware Tools, la creación y comprobación de puntos de restauración, la verificación del estado de activación, el análisis del arranque, la configuración de red, la ejecución de comandos administrativos y la instalación y desinstalación de aplicaciones.

Fase 1. Instalación del sistema operativo

El primer bloque del sprint consistió en crear una nueva máquina virtual en VMware y asociarle la imagen ISO de Windows 11. Durante este proceso se definieron la ubicación de almacenamiento de la máquina virtual y la configuración básica de hardware, incluyendo memoria, procesador y disco virtual. Esta fase permitió disponer de un entorno aislado sobre el que trabajar sin afectar al sistema anfitrión.

Durante la instalación se seleccionó la edición Windows 11 Pro N. A continuación se completó el asistente de instalación y se configuraron los parámetros iniciales del sistema, como idioma, región, distribución de teclado, tipo de uso y cuenta de usuario local. Una vez terminado el proceso, se comprobó que el sistema iniciaba correctamente y que el escritorio era accesible con normalidad.

Una vez finalizada la instalación base, se instalaron las VMware Tools. Esta operación mejoró la integración del sistema invitado con VMware, optimizando la resolución, los controladores y la interacción general con el entorno virtualizado. Tras ello se verificó nuevamente el estado del escritorio y el correcto funcionamiento del sistema.

Fase 2. Puntos de restauración

Después de la instalación se accedió a la herramienta de protección del sistema para habilitar los puntos de restauración. Primero se localizó la utilidad desde el buscador, después se activó la protección sobre la unidad principal y, a continuación, se creó manualmente un punto de restauración. Esta operación permitió fijar un estado del sistema al que volver posteriormente.

Para comprobar el funcionamiento de la restauración del sistema, se introdujo un cambio significativo mediante la instalación masiva de software con Ninite. Se instalaron navegadores, herramientas de comunicación, bibliotecas de ejecución, utilidades de compresión, almacenamiento en la nube y otros programas de uso común. Una vez generado este nuevo estado, se inició el proceso de restauración para volver al punto anterior y se comprobó que el sistema revertía correctamente los cambios realizados.

Fase 3. Licencias de Windows

En la fase de licenciamiento se revisó el estado de activación del sistema desde la configuración de Windows. La comprobación mostró que el sistema no estaba activado. Después se realizó una verificación adicional mediante línea de comandos con slmgr /xpr, lo que permitió comprobar desde una vía técnica el estado real de la licencia.

A partir de esta comprobación se analizó el modelo de licenciamiento de Windows. Se distinguieron las licencias OEM, Retail, por volumen y educativas. En el contexto de una máquina virtual, la opción más coherente es una licencia por volumen o una licencia educativa, ya que la OEM queda vinculada al hardware físico y la Retail, aunque es funcional, resulta menos eficiente si el entorno se reutiliza o replica. También se realizó una valoración económica aproximada del sistema operativo, situando Windows 11 Home en un rango aproximado de 120 a 150 euros y Windows 11 Pro entre 200 y 260 euros en formato Retail.

Fase 4. Gestor de arranque

En este bloque se abrió el símbolo del sistema con privilegios de administrador y se ejecutó el comando bcdedit para analizar la configuración del arranque de Windows. La salida permitió identificar claramente los bloques correspondientes al Administrador de arranque de Windows y al Cargador de arranque de Windows. A partir de estos datos se interpretaron parámetros como el sistema por defecto, el tiempo de espera antes del arranque automático, la partición del sistema operativo y el archivo responsable de su carga.

Del análisis se concluyó que el sistema arrancaba con la entrada actual, que el tiempo de espera estaba fijado en 30 segundos, que Windows se encontraba instalado en la partición C: y que el archivo encargado de iniciar la carga era \Windows\system32\winload.efi. Con ello se documentó la secuencia básica de arranque de Windows y la función diferenciada del Boot Manager y del Boot Loader.

Fase 5. Red básica

La fase de red se centró en la revisión y modificación de la configuración IP del sistema. Se accedió a la configuración del adaptador desde el Panel de control, se comprobó el direccionamiento actual mediante ipconfig y se probaron tanto la configuración automática mediante DHCP como la configuración manual de IP y DNS. Para validar la conectividad se ejecutó un ping a un dominio externo.

Fase 6. Comandos generales

En esta fase se utilizaron PowerShell y la línea de comandos para ejecutar tareas básicas y administrativas. Se abrieron consolas, se listó contenido con dir, se creó una carpeta con mkdir, se generó un archivo de texto mediante echo y posteriormente se eliminó con del. También se probaron comandos del sistema como taskkill, systeminfo, hostname y whoami, así como comandos de red como ipconfig, ping y netstat -an. Por último, se ejecutaron utilidades complementarias como tree, cls, help y shutdown /s /t 0.

Fase 7. Instalación de aplicaciones

El sprint se cerró con la instalación y comprobación de software. Se descargó e instaló Google Chrome desde su fuente oficial, se verificó su funcionamiento y, posteriormente, se instaló una aplicación desde Microsoft Store. Después de comprobar que la aplicación de la Store se ejecutaba correctamente, se procedió a su desinstalación y se verificó que desaparecía del sistema.
