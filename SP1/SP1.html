Sprint 1. Instalación y configuración inicial de Windows en máquina virtual

Volver al inicio Siguiente práctica

Enfoque del sprint

Este sprint se centra en la instalación y configuración inicial de Windows 11 dentro de una máquina virtual, siguiendo un proceso ordenado que permita dejar el sistema operativo correctamente implantado, configurado y listo para su uso. No se trata solo de completar la instalación, sino de entender qué se está haciendo en cada fase y justificar técnicamente las decisiones tomadas.

Además de instalar el sistema, es necesario documentar el proceso con capturas de pantalla colocadas en el punto exacto donde se explica cada acción. De este modo, la práctica no solo muestra el resultado final, sino también la evidencia visual de cada paso importante: creación de la máquina virtual, instalación del sistema, restauración, activación, red, comandos, software base y comprobaciones de arranque.

Objetivos del Sprint 1

  • Instalar Windows 11 en una máquina virtual correctamente configurada.
  • Realizar la configuración inicial del sistema para dejarlo operativo.
  • Crear y comprobar puntos de restauración del sistema.
  • Verificar el estado de activación y justificar el tipo de licencia más adecuado.
  • Analizar el gestor de arranque de Windows y sus parámetros principales.
  • Comprobar la conectividad de red y la configuración IP del entorno virtual.
  • Practicar comandos básicos y de administración en PowerShell y CMD.
  • Instalar y desinstalar aplicaciones verificando su funcionamiento.
  • Documentar todo el procedimiento con explicaciones breves y capturas relevantes.

Fase 1. Instalación del sistema operativo

Paso 1. Crear la máquina virtual

Se crea una máquina virtual en VMware destinada a alojar Windows 11. En este paso se define un entorno aislado que permite realizar toda la práctica sin afectar al sistema anfitrión. Durante el asistente de creación se selecciona la imagen ISO del sistema y se establece la ubicación donde se almacenará la máquina virtual.

Paso 2. Asignar recursos

A continuación se configura el hardware virtual de la máquina. Se asigna una cantidad suficiente de memoria RAM, procesador y almacenamiento para garantizar que Windows 11 pueda instalarse y ejecutarse con normalidad. Esta decisión condiciona tanto el rendimiento de la máquina virtual como el equilibrio con los recursos disponibles en el equipo anfitrión.

Paso 3. Cargar la ISO de Windows 11

Una vez creada la máquina virtual, se asocia la imagen ISO de Windows 11 a la unidad virtual correspondiente para que el sistema pueda arrancar desde ella e iniciar el asistente de instalación. La ISO seleccionada corresponde a Windows 11 Pro N, edición que prescinde de determinados componentes multimedia preinstalados, en línea con la normativa europea de competencia.

Paso 4. Instalar el sistema

Durante la instalación se selecciona la edición concreta del sistema, en este caso Windows 11 Pro N, y se configura el entorno básico de implantación. Esta elección se justifica por su adecuación al contexto académico europeo y por permitir trabajar con el sistema operativo sin determinados componentes propietarios preinstalados.

Paso 5. Comprobar que arranca correctamente

Antes de continuar con el resto de la práctica, se verifica que la máquina virtual está correctamente preparada y que arranca desde la ISO sin incidencias. Una vez completado el proceso, se comprueba que Windows 11 inicia correctamente y alcanza el escritorio con normalidad.

Configuración inicial tras la instalación

Una vez arranca el asistente de configuración inicial, se seleccionan la región, el idioma y la distribución de teclado. Después se elige el tipo de uso previsto para el sistema y se crea una cuenta de usuario local. En este caso, al tratarse de un entorno de prácticas en una máquina virtual aislada, la cuenta se configura sin contraseña. Finalmente, se verifica el primer inicio del sistema y la correcta carga del escritorio de Windows 11.

Instalación de VMware Tools

Para mejorar la integración entre el sistema huésped y VMware, se instalan las VMware Tools. Estas herramientas permiten ajustar automáticamente la resolución, mejorar el rendimiento gráfico, habilitar funcionalidades de integración y optimizar la experiencia de uso general de la máquina virtual.

Fase 2. Puntos de restauración

Paso 6. Buscar “Crear un punto de restauración”

Se localiza la herramienta de restauración desde el buscador del sistema. Este paso permite acceder a la función de protección del sistema, imprescindible para generar puntos de restauración que permitan revertir cambios posteriores.

Paso 7. Activar la protección del sistema en el disco C:

Una vez abierta la ventana de propiedades del sistema, se activa la protección sobre la unidad principal. Esta configuración permite que Windows almacene estados previos del sistema que puedan recuperarse si una instalación o cambio provoca incidencias.

Paso 8. Crear un punto de restauración manual

Tras activar la protección, se genera manualmente un punto de restauración. Esta acción crea una referencia a la que podrá volver el sistema más adelante, lo que resulta especialmente útil antes de realizar instalaciones masivas o modificaciones de configuración.

Paso 9. Hacer un cambio en el sistema

Para comprobar el funcionamiento real del punto de restauración, se realiza un cambio significativo en el sistema mediante la instalación masiva de aplicaciones con Ninite. Entre el software instalado se incluyen navegadores, herramientas de comunicación, utilidades de compresión, bibliotecas de ejecución, herramientas de acceso remoto, almacenamiento en la nube y otros programas de uso habitual. Esto permite generar un estado claramente diferente que después podrá revertirse.

Paso 10. Restaurar y comprobar

Una vez introducidos los cambios, se inicia el proceso de restauración para volver al punto previamente creado. El asistente de restauración permite seleccionar el punto disponible, revisar los cambios afectados y ejecutar la reversión. Finalmente, se comprueba que el sistema vuelve al estado anterior y que los programas añadidos desaparecen o dejan de estar presentes como consecuencia de la restauración.

Fase 3. Licencias de Windows

Paso 11. Abrir Configuración → Sistema → Activación

Se accede al apartado de activación de Windows para comprobar el estado de la licencia del sistema. Esta pantalla constituye la referencia principal para verificar si el sistema está activado o no desde la interfaz gráfica.

Paso 12. Ver si Windows está activado

La comprobación confirma que Windows no está activado en esta instalación. Esto implica que el sistema puede utilizarse para prácticas, pero no dispone de una licencia validada de forma completa, lo que limita determinados aspectos de personalización y regularidad de uso.

Paso 13. Ejecutar en CMD slmgr /xpr

Para complementar la verificación visual, se realiza una comprobación técnica mediante línea de comandos. El comando slmgr /xpr permite comprobar si la activación del sistema es permanente o si no existe una licencia válida asociada a la instalación actual.

Paso 14. Analizar el licenciamiento de Windows y explicarlo brevemente

En este apartado se analiza el modelo de licenciamiento de Windows con el objetivo de determinar qué modalidad resulta más adecuada en un entorno de virtualización. Windows dispone de distintos tipos de licencia. La licencia OEM queda vinculada al hardware físico del equipo y no puede transferirse libremente. La licencia Retail ofrece mayor flexibilidad y permite reutilización en otros equipos, aunque está orientada a escenarios individuales. En entornos corporativos o institucionales se emplean licencias por volumen, mientras que las licencias educativas se adaptan a contextos académicos.

En una máquina virtual no existe una vinculación directa y estable con un hardware físico concreto. Por ello, una licencia OEM no resulta la opción más adecuada. La licencia Retail sería funcional, pero poco eficiente si se pretende reutilizar o replicar el entorno. En la práctica, las opciones más coherentes para este caso serían licencias por volumen o educativas, ya que permiten mayor flexibilidad, reutilización y adaptación a laboratorios o entornos docentes.

Paso 15. Consultar el precio aproximado de una licencia de Windows

Además del análisis técnico del licenciamiento, se realiza una aproximación al coste económico del sistema operativo. De forma orientativa, una licencia Windows 11 Home suele situarse aproximadamente entre 120 y 150 euros, mientras que Windows 11 Pro suele situarse entre 200 y 260 euros. Estos valores corresponden habitualmente a licencias Retail. Las licencias OEM suelen ser más económicas, pero quedan vinculadas al hardware. Por su parte, las licencias por volumen o educativas no suelen disponer de un precio unitario público, ya que dependen de acuerdos concretos con organizaciones o instituciones.

Esto permite contextualizar que la implantación de un sistema operativo no solo tiene una dimensión técnica, sino también económica, especialmente cuando se plantea en múltiples equipos o en infraestructuras virtualizadas.

Fase 4. Gestor de arranque

Paso 16. Abrir Command Prompt como administrador

Para analizar la configuración del arranque se abre el símbolo del sistema con privilegios de administrador. Este nivel de permisos es necesario para consultar información sensible del proceso de inicio del sistema.
