Sprint 1. Instalación y configuración inicial de Windows en máquina virtual

Volver al inicio Siguiente práctica

Enfoque del sprint

Este sprint queda orientado a la implantación de un sistema Windows dentro de una máquina virtual, documentando tanto la instalación como la configuración inicial necesaria para dejar el entorno operativo, conectado y preparado para trabajar.

El trabajo no se limita a instalar Windows: también hay que justificar la licencia elegida, configurar red, instalar software base, comprobar el arranque y dejar evidencia clara de todo el proceso mediante capturas y explicaciones breves.

Objetivos del Sprint 1

  • Identificar los elementos funcionales principales de un sistema operativo Windows.
  • Realizar la instalación y configuración inicial del sistema en una máquina virtual.
  • Configurar y verificar los servicios y protocolos básicos de red.
  • Instalar el software base necesario para el uso y mantenimiento del sistema.
  • Aplicar técnicas de recuperación o reparación ante una incidencia de arranque.
  • Documentar el procedimiento completo con evidencias visuales y conclusiones.

Fase 1. Preparación de la máquina virtual

  1. Crear una máquina virtual destinada a alojar Windows.
  2. Definir la configuración de hardware virtual: memoria RAM, procesador, disco y adaptador de red.
  3. Cargar la ISO de instalación de Windows.
  4. Justificar por qué esa configuración es válida para un entorno de ofimática o trabajo colaborativo.
  5. Registrar capturas de la creación de la VM, recursos asignados y disco virtual generado.

Qué conviene explicar: tipo de máquina virtual, recursos elegidos, tamaño del disco y motivo de la configuración.

Fase 2. Instalación de Windows e identificación de elementos funcionales

  1. Ejecutar la instalación completa del sistema operativo.
  2. Configurar idioma, región, usuario, contraseña y opciones iniciales.
  3. Comprobar que el sistema inicia correctamente tras la instalación.
  4. Identificar durante el proceso o después del primer arranque los elementos funcionales del sistema.
  5. Describir brevemente el gestor de archivos, el gestor de procesos, el sistema de permisos, el registro de Windows y las opciones de configuración inicial.

Capturas recomendadas: asistente de instalación, primer inicio de sesión, Explorador de archivos, Administrador de tareas, registro o paneles de configuración relevantes.

Fase 3. Licenciamiento de Windows

  1. Revisar el estado de activación del sistema.
  2. Consultar el tipo de licencia que se está utilizando o la que se simularía en el proyecto.
  3. Explicar las diferencias básicas entre licencias OEM, Retail, Volumen y Educacional.
  4. Indicar cuál encaja mejor en este entorno virtual y justificar la elección.
  5. Añadir una comprobación del sistema como slmgr /xpr o la pantalla de activación de Windows.

Importante: aquí no solo interesa comprobar si está activado, sino argumentar qué modalidad de licencia tendría sentido en una implantación educativa o profesional.

Fase 4. Configuración de red y conectividad

  1. Configurar la conectividad de la máquina virtual.
  2. Comprobar la configuración IP con herramientas como ipconfig.
  3. Verificar el funcionamiento de DHCP o configurar una IP estática si se pide en la práctica.
  4. Revisar parámetros como puerta de enlace, DNS y grupo de trabajo.
  5. Realizar pruebas de conectividad con ping y otras utilidades básicas.

Capturas recomendadas: configuración del adaptador de red, salida de ipconfig, pruebas de ping y resultado final de la conexión.

Fase 5. Instalación de software base y seguridad

  1. Instalar las actualizaciones necesarias del sistema.
  2. Revisar el antivirus y el firewall de Windows o añadir software equivalente si la práctica lo requiere.
  3. Instalar herramientas básicas: compresión, navegador alternativo y utilidades de administración.
  4. Comprobar que las aplicaciones instaladas funcionan correctamente.
  5. Dejar constancia del software añadido y de su utilidad dentro del sistema.

Ejemplos de software base: navegador alternativo, compresor, utilidades de mantenimiento, herramientas de actualización o diagnóstico.

Fase 6. Incidencia de arranque y recuperación

  1. Simular o resolver una incidencia relacionada con el arranque del sistema.
  2. Acceder a las opciones avanzadas de recuperación de Windows o al entorno WinRE.
  3. Usar, si procede, herramientas como reparación de inicio, bcdedit o bootrec.
  4. Explicar qué problema se provoca o detecta y qué pasos se siguen para solucionarlo.
  5. Verificar que el sistema vuelve a arrancar con normalidad.

Lo importante en esta parte: demostrar criterio técnico. Aunque la incidencia sea sencilla, debe quedar claro qué herramienta se ha usado y por qué.

Fase 7. Documentación final

  1. Ordenar todas las capturas siguiendo el proceso real del sprint.
  2. Explicar de forma breve cada paso importante.
  3. Indicar incidencias detectadas, decisiones tomadas y ajustes realizados.
  4. Incluir una conclusión final sobre el resultado de la implantación.
  5. Relacionar el trabajo con el uso de virtualización: hardware virtual, discos, snapshots o ajustes necesarios.

Recomendación: cada captura debe aportar una evidencia concreta. No hace falta llenar la práctica de imágenes repetidas; es mejor que cada una demuestre una acción o una comprobación real.

Criterios de evaluación trabajados

  • 1.1 Identificación de los elementos funcionales de un sistema informático.
  • 1.2 Identificación de características, funciones y arquitectura del sistema operativo.
  • 1.3 Comparación de versiones y tipos de licencias según requisitos y contexto de uso.
  • 1.5 Aplicación de técnicas de actualización y rescate del sistema.
  • 1.6 Resolución de incidencias del sistema y del proceso de inicio.
  • 1.7 Uso de herramientas para conocer el software instalado y su procedencia.
  • 1.8 Elaboración de documentación de soporte sobre instalación e incidencias.
  • 2.4 Instalación, configuración y verificación de protocolos de red.
  • 2.7 Uso de máquinas virtuales para realizar tareas de configuración y análisis.

Guía rápida de evidencias

  • Creación y configuración de la máquina virtual.
  • Instalación de Windows paso a paso.
  • Primer arranque y configuración inicial.
  • Comprobación de activación o tipo de licencia.
  • Configuración y pruebas de red.
  • Instalación del software base.
  • Proceso de reparación o análisis del arranque.
  • Resultado final del sistema operativo funcionando correctamente.
