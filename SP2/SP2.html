Sprint 2. Usuarios, almacenamiento, procesos y recuperación en Windows

Volver al inicio Siguiente práctica

Enfoque del sprint

Este sprint se centra en la administración de Windows ya instalado en máquina virtual. El objetivo es dejar el sistema preparado para trabajar con usuarios y grupos, políticas locales, almacenamiento adicional, cuotas, procesos, copias de seguridad y mecanismos básicos de recuperación.

A partir del enunciado y de la propuesta práctica, el sprint queda orientado a una implementación realista en la que se añaden discos virtuales, se organizan particiones, se limitan recursos por usuario y se automatizan tareas de copia y optimización del entorno.

Objetivos del Sprint 2

  • Crear y configurar cuentas de usuario locales, grupos y políticas de contraseñas.
  • Gestionar procesos y servicios de Windows según las necesidades del sistema.
  • Administrar discos, particiones, volúmenes y sistemas de archivos.
  • Aplicar cuotas de disco para controlar el espacio asignado a los usuarios.
  • Implementar copias de seguridad y automatización básica mediante scripts.
  • Configurar permisos avanzados con ACLs y comprobar excepciones de acceso.
  • Documentar la estructura del sistema, las pruebas realizadas y la recuperación ante incidencias.

Fase 1. Preparación del almacenamiento

  1. Añadir un nuevo disco virtual a la máquina virtual Windows.
  2. Abrir Administración de discos e inicializar el nuevo disco.
  3. Crear dos particiones o volúmenes: Dades en NTFS y Portable en FAT32.
  4. Asignar letras de unidad y verificar la configuración con diskpart si se desea comprobar por consola.
  5. Explicar por qué se usan sistemas de archivos distintos y qué ventajas tiene cada uno.

Qué debe quedar claro: diferencia entre NTFS, FAT32 y exFAT, uso previsto de cada volumen y evidencia de que el almacenamiento quedó correctamente configurado.

Fase 2. Usuarios, grupos y políticas locales

  1. Crear dos usuarios locales, por ejemplo alumne1 y alumne2.
  2. Crear un grupo nuevo, por ejemplo Limitats, y añadir a ambos usuarios.
  3. Revisar o configurar políticas locales de contraseñas y directivas básicas de seguridad.
  4. Comprobar que la pertenencia al grupo afecta correctamente a los permisos asignados.
  5. Documentar el proceso con capturas de usuarios, grupos y directivas locales utilizadas.

Aquí se trabaja: creación de cuentas, pertenencia a grupos, control de acceso y endurecimiento básico del sistema mediante directivas locales.

Fase 3. Cuotas de disco

  1. Activar cuotas de disco en la partición Dades, que debe estar en NTFS.
  2. Establecer un límite por usuario, por ejemplo 300 MB, con nivel de advertencia.
  3. Iniciar sesión con los usuarios creados y copiar archivos en la unidad para probar la cuota.
  4. Comprobar qué ocurre al superar el límite configurado.
  5. Guardar evidencias del límite aplicado y del bloqueo o advertencia producidos.

Objetivo técnico: demostrar que el sistema controla el espacio disponible por usuario y que la partición adecuada para ello es NTFS.

Fase 4. Copias de seguridad y automatización

  1. Añadir un tercer disco virtual y formatearlo en NTFS como unidad de copias, por ejemplo Backups.
  2. Crear una carpeta como E:\CopiasUsuarios.
  3. Preparar un script .bat que copie C:\Users\%USERNAME% a la unidad de copias.
  4. Asignar el script al inicio de sesión desde gpedit.msc o la configuración equivalente.
  5. Comprobar con alumne1 y alumne2 que la copia se ejecuta correctamente al iniciar sesión.

Ejemplo de objetivo: automatizar una copia sencilla del perfil del usuario y justificar su utilidad en entornos educativos o de laboratorio.

Fase 5. Gestión de procesos y servicios

  1. Listar los procesos activos con herramientas como tasklist o el Administrador de tareas.
  2. Guardar la salida en un archivo para usarla como evidencia documental.
  3. Identificar procesos prescindibles para un entorno virtual o con pocos recursos, como aplicaciones de sincronización o mensajería no necesarias.
  4. Finalizar de forma controlada algunos procesos no esenciales con taskkill.
  5. Añadir esas órdenes al script de inicio de sesión si se quiere automatizar la optimización del sistema.

Conviene explicar: diferencia entre proceso crítico y proceso prescindible, y qué mejora se busca en una máquina virtual al reducir carga innecesaria.

Fase 6. Permisos avanzados y ACLs

  1. Crear una carpeta de trabajo, por ejemplo D:\Projectes.
  2. Romper la herencia si hace falta y dejar únicamente los permisos deseados.
  3. Dar control total al grupo Limitats sobre la carpeta.
  4. Comprobar con alumne1 que puede crear, modificar y borrar archivos.
  5. Aplicar una excepción para alumne2 con solo lectura, por ejemplo usando icacls.
  6. Verificar que alumne2 puede leer, pero no modificar ni crear archivos.
  7. Consultar los permisos finales para dejar constancia de la ACL aplicada.

Lo importante aquí: mostrar que las ACLs permiten un control más fino que un permiso general de grupo y que una excepción concreta puede limitar a un usuario determinado.

Fase 7. Estructura del sistema y sistemas de archivos

  1. Comparar los sistemas de archivos NTFS, FAT32 y exFAT.
  2. Describir la estructura básica de directorios de Windows.
  3. Identificar ubicaciones importantes como C:\Windows, C:\Windows\System32, C:\Program Files y C:\Users.
  4. Relacionar cada ubicación con su función dentro del sistema operativo.
  5. Documentar también los archivos o herramientas de configuración que resulten relevantes en la práctica.

Finalidad: no quedarse solo en la parte práctica de administración, sino comprender dónde residen los componentes principales del sistema y qué papel cumple cada directorio.

Fase 8. Copia de seguridad y recuperación ante desastres

  1. Configurar un mecanismo de copia o restauración del sistema según la práctica realizada.
  2. Usar puntos de restauración, historial de archivos o una herramienta de copia de Windows, según lo que se haya implementado.
  3. Simular una incidencia sencilla o una pérdida de datos controlada.
  4. Comprobar que la restauración o recuperación funciona correctamente.
  5. Explicar de forma ordenada el procedimiento seguido y el resultado final.

Esta parte debe cerrar el sprint: no solo administráis recursos, también dejáis preparado un método básico de recuperación.

Verificación final

  • Los usuarios y grupos se crean correctamente y heredan los permisos esperados.
  • Las cuotas bloquean o advierten cuando se supera el espacio permitido.
  • Las particiones y letras de unidad aparecen correctamente en Windows.
  • El script de copia se ejecuta al iniciar sesión.
  • Los procesos prescindibles pueden cerrarse manual o automáticamente.
  • Las ACLs distinguen entre permisos de grupo y excepciones individuales.
  • Existe evidencia de una copia o restauración válida ante una incidencia.

  • 2.8 Documentación de tareas de configuración del software de base.
  • 3.1 Comparación de sistemas de archivos y sus ventajas.
  • 3.2 Descripción de la estructura de directorios del sistema operativo.
  • 3.3 Identificación de directorios que contienen binarios, órdenes y librerías del sistema.
  • 3.4 Uso de herramientas de administración de discos para particiones y volúmenes.
  • 3.8 Documentación de operaciones realizadas y métodos de recuperación ante desastres.

Guía rápida de evidencias

  • Discos añadidos, inicialización y particiones creadas.
  • Usuarios, grupo Limitats y políticas locales.
  • Configuración y prueba de cuotas en Dades.
  • Script de copia y ejecución automática al iniciar sesión.
  • Listado y cierre de procesos no esenciales.
  • Permisos ACL sobre D:\Projectes con excepción para alumne2.
  • Comparativa de sistemas de archivos y estructura de directorios.
  • Prueba final de restauración o recuperación.
