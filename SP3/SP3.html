Sprint 3. Administración de dominios y seguridad

Objetivo: instalar Active Directory Domain Services (AD DS), promover el servidor a controlador de dominio y unir un cliente Windows 11 al dominio.

Paso 1. Comprobación de red

Captura 1.

Se revisa la IP del equipo cliente para verificar su configuración de red inicial.

Captura 2.

Se comprueba la IP del servidor para confirmar que cliente y servidor están en la misma red.

Paso 2. Instalación de AD DS

Captura 3.

Desde Server Manager se inicia "Agregar roles y características".

Captura 4.

Se elige instalación basada en roles o características.

Captura 5.

Se selecciona el servidor de destino donde se aplicará el rol.

Captura 6.

Se marca Active Directory Domain Services (AD DS).

Captura 7.

Se revisa el resumen y se completa la instalación del rol.

Paso 3. DNS del cliente y promoción del dominio

Captura 8.

En el cliente se configura como DNS preferido la IP del servidor (172.16.147.129).

Captura 9.

Se ejecuta la promoción del servidor a controlador de dominio creando un nuevo bosque.

Captura 10.

Se define la contraseña DSRM y se continúa con la validación de requisitos.

Paso 4. Usuarios y Unidad Organizativa

Captura 11.

Se abre "Usuarios y equipos de Active Directory" para administrar objetos del dominio.

Captura 12.

Se inicia el asistente para crear un nuevo usuario de dominio.

Captura 13.

Se introducen los datos del usuario (nombre e identificador de inicio de sesión).

Captura 14.

Se define contraseña y políticas básicas de cuenta para finalizar el alta.

Captura 15.

Se inicia la creación de una Unidad Organizativa (OU).

Captura 16.

Se asigna nombre a la OU y se confirma su creación.

Captura 17.

Se verifica que la OU aparece correctamente en el árbol del dominio.

Paso 5. Unión del cliente al dominio y validación

Captura 18.

En Windows 11 se cambia la pertenencia de grupo de trabajo a dominio.

Captura 19.

Se introducen credenciales con permisos para unir el equipo al dominio.

Captura 20.

Se confirma el mensaje de unión correcta al dominio.

Captura 21.

Tras reiniciar, se prepara el inicio de sesión con cuenta de dominio.

Captura 22.

Se valida el acceso con el usuario de Active Directory en el cliente.

Nota final sobre edición Evaluation

Si Windows Server está en edición Evaluation, puede tener limitaciones periódicas. Para un entorno estable, convierte la edición a Standard/Datacenter y activa con licencia válida mediante métodos oficiales de Microsoft.

Entorno de trabajo: Windows Server Datacenter 2025 y Windows 11 Pro.
